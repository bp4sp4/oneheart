응답 형식 및 점수 계산
--------------------
- 각 문항 응답은 5점 척도(내부 값): -2, -1, 0, 1, 2
- 일부 문항은 '반전(reverse scoring)'으로 설정되어 있습니다. 반전 문항은 선택값에 마이너스 부호를 붙여 합산합니다.
- 최종 점수는 모든 문항의(반전 적용 후) 합입니다.

반전 항목(중요)
----------------
- 반전은 이미 기획 단계에서 정해졌습니다. 반전된 문항 번호(사람이 보는 기준 1-based)는 다음과 같습니다:
  - A축(1–25): [2,4,6,7,9,12,14,16,18,20,22,23]
  - B축(26–50): [27,29,31,33,35,37,39,41,43,45,47,48,50]
  - C축(51–75): [52,54,56,58,60,62,66,68,70,72,74]
  - D축(76–100): [77,79,81,83,85,87,89,92,94,96,98,100]

주의 — 절대 바뀌면 안 되는 것들

1. 문항 순서: 문항의 순서를 바꾸면 반전 설정·점수 계산이 모두 어긋납니다. 순서는 고정하세요.
2. 응답 값 스케일: 현재 스케일은 반드시 [-2,-1,0,1,2]입니다. 변경 시 전체 규칙 재검증이 필요합니다.
3. 반전 처리 방식: 현재는 값의 부호 반전 방식입니다. 다른 방식으로 바꾸면 결과 해석이 달라집니다.


간단한 실행 방법(개발용)
----------------------
1. 의존성 설치:
```bash
npm install
```
2. 개발 서버 실행:
```bash
npm run dev
```
3. 브라우저에서 앱 확인: `http://localhost:3000`

변경 및 요청 방법
-----------------
- 반전 항목을 변경하고 싶으면 '문항 번호(1-based)' 목록을 주세요. 개발자가 0-based로 변환해 적용합니다.
- 문항 텍스트 수정은 가능하지만 '순서 변경'은 불허합니다.
- 점수→유형 매핑(어떤 점수대가 어떤 유형인지)은 별도 기획(데이터 기반 권장)입니다.

다음 단계 제안
----------------
1. 문항 텍스트(100개) 최종 확정 → 저는 이를 `app/page.tsx`에 반영합니다.
2. 점수→유형 매핑(임계값) 정하기 → 테스트 데이터로 검증 후 적용합니다.
3. 이메일 발송(서버 연동) 또는 PDF 다운로드 기능 추가.

# Mother Type Test — Next.js 샘플 프로젝트 (oneheart)

이 프로젝트는 엄마 성향 테스트(나중에 질문 리스트 추가 예정)를 위한 Next.js + TypeScript 기본 골격입니다.

빠른 시작

```bash
cd ~/Documents/GitHub/oneheart
npm install
npm run dev
```
환경파일 작성 방법
-----------------
- 프로젝트 루트에 `.env.local` 파일을 생성하고 위 값을 넣으세요. 예:
```
BREVO_API_KEY=REPLACE_WITH_YOUR_KEY
BREVO_SENDER_EMAIL=no-reply@example.com
BREVO_SENDER_NAME=엄마유형테스트
```
- 로컬에서 테스트할 때는 `.env.local`에 실제 Brevo API 키를 넣고 서버를 재시작하세요.

- `pages/api/mother` : 예시 API 응답 (Type: `MotherResponse`)
- `types/mother.ts` : `MotherResponse` 타입 정의
- `pages/index.tsx` : 예제 UI (결과 불러오기, JSON 다운로드)

계속할 작업
- 문항 리스트 스키마 추가 및 점수 계산 로직 구현
- 점수 기반 유형 매핑 및 결과 생성
- 이메일 전송 및 공유 기능 추가
